# SOC Incident Detection & Response Using Windows Event Log

## Overview
This project demonstrates a simulated Security Operations Center (SOC) workflow focused on detecting,
investigating, and responding to security incidents using SIEM concepts and industry-standard frameworks.

## Objectives
- Monitor and analyze security events using SIEM
- Perform alert triage and log correlation
- Identify indicators of compromise (IOCs)
- Map attacker behavior to MITRE ATT&CK
- Document incidents following NIST Incident Response lifecycle

## Tools & Technologies
- Splunk SIEM (Academic / Free)
- Windows Event Logs
- Linux Logs
- MITRE ATT&CK Framework
- NIST Incident Response Lifecycle

## Incident Scenario
Simulated brute-force login attempts followed by suspicious PowerShell execution indicating potential
privilege escalation activity.

## Investigation Process
- Analyzed authentication and process execution logs
- Correlated events across multiple sources
- Identified suspicious IP addresses and user behavior
- Mapped techniques to MITRE ATT&CK

## Response & Documentation
- Created incident timeline
- Assessed impact and risk level
- Recommended containment and remediation steps
- Documented findings in a professional incident report

## Outcome
This project demonstrates hands-on SOC Tier 1 skills including alert validation, investigation,
documentation, and escalation readiness.
